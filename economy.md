## 1. Theme Store (Маркетплейс Дизайн-Тем)

### Концепція

Маркетплейс, де дизайнери та AI-генератори створюють і продають Color Schemes (CSS Root Themes).
Кожна тема — це набір CSS-змінних, що миттєво застосовуються до Sandbox та всіх компонентів.

### Комісія: **18%**

| Учасник                 | Частка | Опис                                            |
| ----------------------- | ------ | ----------------------------------------------- |
| **Автор теми**          | 82%    | Дизайнер або AI-генератор, який створив тему    |
| **NaN0Web** (платформа) | 18%    | Хостинг, модерація, інфраструктура маркетплейсу |

```yaml
  --ib-blue: '#0d6efd'

```

## 2. AI Proxy з Кешуванням (Детальна Архітектура)

### 2.1. Проблема

У моделі BYOK (див. `packages/ui-voice/architecture.md` §9) користувач сам платить за кожен токен до LLM.
Але значна частина запитів у Sandbox — це **типові, повторювані інтенти**:

| Тип запиту        | Приклад                              | Частота  |
| ----------------- | ------------------------------------ | -------- |
| Зміна теми        | "Змінити тему на темну"              | ~25%     |
| Зміна мови        | "Перемкни на англійську"             | ~15%     |
| Reset             | "Скинь до дефолту"                   | ~10%     |
| Додати варіацію   | "Додай ще одну кнопку"               | ~20%     |
| Прості Props      | "Зроби радіус більше"                | ~15%     |
| **Разом типових** |                                      | **~85%** |
| Складні/унікальні | "Згенеруй палітру для luxury-бренду" | ~15%     |

**85% запитів** не потребують звернення до LLM.

### 2.2. Архітектура кешування

```
Запит користувача
      │
      ▼
┌──────────────┐
│ Intent       │  Regex/Pattern Matcher (Level 0 — локальний)
│ Classifier   │  → Якщо збіг: виконати ui-api команду, повернути кешовану відповідь
│              │  → Якщо незбіг: передати далі
└──────┬───────┘
       │ (тільки ~15% запитів проходять далі)
       ▼
┌──────────────┐
│ Semantic     │  Embedding-based кеш (Level 1 — edge/server)
│ Cache        │  → Якщо семантично схожий запит вже є: повернути кешовану відповідь
│              │  → Якщо ні: передати до LLM
└──────┬───────┘
       │ (тільки ~5% оригінальних запитів)
       ▼
┌──────────────┐
│ LLM API      │  Реальний виклик до Cerebras/Groq/OpenAI (Level 2 — cloud)
│ (BYOK key)   │  → Відповідь зберігається в кеш для майбутніх запитів
└──────────────┘
```

### 2.3. Три Рівні Кешу

**Level 0 — Intent Pattern Matcher (Локальний, $0)**

- Працює на клієнті (в браузері).
- Regex + keyword-бот. Не потребує мережі.
- Кешує ~85% стандартних команд.

```js
const PATTERNS = [
  { match: /тем[уа].*(темн|dark)/i, action: 'theme.set', params: { theme: 'dark' } },
  { match: /тем[уа].*(світл|light)/i, action: 'theme.set', params: { theme: 'light' } },
  { match: /(мов[уа]|lang).*(en|англ)/i, action: 'locale.set', params: { locale: 'en' } },
  { match: /(скинь|reset|undo)/i, action: 'sandbox.reset', params: {} },
  { match: /(додай|add).*(варіац|variation)/i, action: 'sandbox.addVariation', params: {} },
]
```

**Level 1 — Semantic Cache (Edge/Server, ~$0.001/запит)**

- Embedding-based: порівнюємо вектор нового запиту з уже збереженими.
- Якщо cosine similarity > 0.92 → повертаємо кешовану відповідь.
- Вартість: тільки embedding запиту (~$0.001/запит через OpenAI text-embedding).
- База кешу зростає з часом → якість кешування покращується.

**Level 2 — LLM (Cloud, BYOK)**

- Повноцінний запит до обраного LLM-провайдера.
- Відповідь одразу зберігається в Level 1 кеш.
- Вартість: ~$0.005-0.02/запит (залежить від провайдера).

### 2.4. Фінансова Модель AI Proxy

#### Варіант A: Продаж як Premium Feature (без маржі на токенах)

| Тір       | Ціна        | Кешування                          | LLM     |
| --------- | ----------- | ---------------------------------- | ------- |
| Free      | $0          | Level 0 (локальний regex)          | Без LLM |
| Developer | $9/мо       | Level 0 + Level 1 (semantic cache) | BYOK    |
| Team      | $29/мо/юзер | Level 0 + Level 1 + shared cache   | BYOK    |

Дохід: від підписки. Кешування — як value-add для платних тірів.

#### Варіант B: AI Proxy з маржою (агрегатор)

NaN0Web виступає **проксі між користувачем та LLM API**, закупаючи токени оптом.

| Параметр                                                    | Значення                     |
| ----------------------------------------------------------- | ---------------------------- |
| Наша закупка (Cerebras, оптом)                              | ~$0.40 / 1M токенів          |
| Продаж користувачу через proxy                              | ~$0.80 / 1M токенів          |
| **Маржа**                                                   | **50%** ($0.40 / 1M токенів) |
| Середній запит у Sandbox                                    | ~500 токенів (input+output)  |
| Собівартість 1 запиту                                       | **$0.0002**                  |
| Ціна 1 запиту для юзера                                     | **$0.0004**                  |
| Запитів на місяць (100 юзерів × 50 запитів/день)            | ~150,000                     |
| Запитів після кешування L0+L1 (тільки ~15% доходять до LLM) | ~22,500                      |
| **Наша маржа/місяць**                                       | 22,500 × $0.0002 = **$4.50** |

⚠️ **Висновок по Варіанту B:** При малих обсягах маржа мінімальна ($4.50/мо).
AI Proxy з маржою має сенс тільки для великих обсягів (10,000+ юзерів).
**Для стартового етапу рекомендую Варіант A (підписка).**

#### Варіант C: Гібрид (рекомендований)

| Тір        | Підписка    | AI Credits                                  | LLM Access                       |
| ---------- | ----------- | ------------------------------------------- | -------------------------------- |
| Free       | $0/мо       | 100 безкоштовних запитів/мо                 | Через наш proxy (L0+L1 кеш)      |
| Developer  | $9/мо       | 2,000 запитів/мо (включено) + BYOK fallback | L0+L1+L2 через proxy або BYOK    |
| Team       | $29/мо/юзер | 10,000 запитів/мо/юзер + BYOK               | L0+L1+L2 + shared team cache     |
| Enterprise | Custom      | Unlimited                                   | Self-hosted proxy, приватний LLM |

**Логіка:**

1. Юзер отримує **включені AI Credits** у підписці (ми оплачуємо з маржі підписки).
2. Коли credits вичерпано — автоматичний fallback на **BYOK** (юзер платить напряму).
3. Кешування L0+L1 знижує реальне споживання на 85% → більшість юзерів не вичерпають credits.
