# Інструкції з використання інтенцій в OLMUI Generator Runners

Цей документ надає детальний посібник для розробників щодо використання **інтенцій** в додатках OLMUI. Інтенції є основним механізмом для керування взаємодіями з користувачем, рендерингом UI, логуванням та обробкою результатів у робочих процесах на основі генераторів. Вони генеруються (yield) з функцій-генераторів у ваших моделях додатків і обробляються функцією `runGenerator` у `GeneratorRunner.js`.

## Огляд шаблону генератора

Додатки OLMUI використовують **функції-генератори** (ES6-генератори з `function*`) для визначення асинхронних, інтерактивних робочих процесів. Точка входу — це клас або функція з методом `run(options)`, який повертає ітератор. Цей ітератор генерує **інтенції** (об'єкти, що описують дії) і отримує відповіді через `next(value)`.

### Ключові концепції

- **Модель точки входу додатка (App Entry Model)**: Верхньорівневий клас (наприклад, `AppEntryModel`), який перелічує команди або підмоделі. Використовується в `runApp` для вибору та виконання команди.
- **Команда/Модель**: Клас або об'єкт з методом-генератором `run(options)`. Створіть екземпляр або передайте безпосередньо в `runGenerator`.
- **Адаптер**: CLI/UI-адаптер (наприклад, `InputAdapter`), який обробляє виконання інтенцій. Надає методи на кшталт `askIntent`, `progressIntent` тощо.
- **Опції (options)**: Передаються в `run()`, включаючи `t` (функцію перекладу) та `db` (екземпляр бази даних).
- **Скасування**: Використовуйте `CancelError` для обробки скасувань користувачем. Генератор може перехоплювати та поширювати їх.
- **Циклювання**: `runApp` циклічно виконується нескінченно, доки не буде повне виходу (наприклад, скасування на основному рівні).

### Базова структура генератора

```javascript
// Приклад класу моделі
class MyCommand {
  async *run(options = {}) {
    // Тут генеруються інтенції
    const response1 = yield { type: 'ask', field: 'name', ... };

    // Обробка response1
    yield { type: 'show', message: `Hello, ${response1.value}!` };

    // Генерація результату
    return { data: 'Final result' }; // Це стає res.value при завершенні
  }
}

// Використання в runApp (фрагмент)
const commandToRun = new MyCommand();
await runGenerator(commandToRun, adapter, options);
```

- **Генерація інтенцій**: Використовуйте `yield intentObject` для паузи виконання та запиту дії.
- **Отримання відповідей**: Наступний `yield` або `return` отримує результат попередньої інтенції через `next(responseValue)`.
- **Помилки**: Генератори можуть використовувати `iter.throw(error)` для поширення винятків. Скасування обробляються спеціально.
- **Завершення**: `return value` завершує генератор; `runGenerator` повертає `{ success: true, data: value, cancelled: false }`.

Якщо модель вже є ітератором (не метод `run()`), передайте його безпосередньо.

## Типи інтенцій

Інтенції — це звичайні об'єкти з обов'язковою властивістю `type`. Додаткові властивості залежать від типу. Адаптер обробляє їх через switch-case у `runGenerator`. Необроблені інтенції передаються як `nextVal`.

### 1. `ask` - Запит введення користувача

Запитує введення користувача (наприклад, текст, вибір, валідація). Обробляється `adapter.askIntent(intent)`.

- **Призначення**: Збір даних інтерактивно в CLI/UI (наприклад, запитання, меню, форми).
- **Обов'язкові властивості**:
  - `type: 'ask'`
  - `field: string` - Унікальний ідентифікатор для введення (наприклад, 'username'). Використовується для зберігання/отримання значень.
- **Необов'язкові властивості**:
  - `schema: Object|Class` - Схема валідації (наприклад, JSON Schema або клас моделі). Якщо `model: true`, трактує `schema` як повну модель для під-виборів.
  - `model: boolean` - Якщо true, `schema` — це модель додатка для вибору команди (використовується в `runApp`).
  - `label: string` - Мітка/промпт для відображення (перекладається через `options.t`).
  - `placeholder: string` - Підказка для введення.
  - `default: any` - Значення за замовчуванням.
  - `validate: Function(value): Promise<string|true>` - Кастомний валідатор. Повертає повідомлення про помилку або `true`.
  - `choices: Array` - Для меню вибору (наприклад, `{ value: 'opt1', label: 'Option 1' }`).
  - `type: string` - Тип введення (наприклад, 'text', 'number', 'select', 'multiselect', 'password').
  - `required: boolean` - Примусове не-порожнє введення.
  - `retry: boolean|number` - Повтор на недійсному введенні (за замовчуванням: true; число обмежує спроби).

- **Відповідь**: `{ value: any, cancelled: boolean }`. Якщо скасовано, кидає `CancelError`.
- **Приклад**:
  ```javascript
  const userInput = yield {
    type: 'ask',
    field: 'username',
    label: 'Enter your username',
    schema: { type: 'string', minLength: 3 },
    validate: async (val) => val.length >= 3 ? true : 'Too short',
  };
  // userInput.value — введене значення
  ```
- **У виборі додатка**: Для головного меню використовуйте `{ type: 'ask', field: 'app', model: true, schema: AppEntryModel }`. `AppEntryModel` повинен експортувати статичні методи або властивості на кшталт `commands: Array<{ name, run }>`.

### 2. `progress` - Індикатор прогресу

Показує/оновлює індикатор прогресу або статус. Обробляється `adapter.progressIntent(intent)`.

- **Призначення**: Відображення тривалої роботи (наприклад, обробка файлів, API-запити).
- **Обов'язкові властивості**:
  - `type: 'progress'`
- **Необов'язкові властивості**:
  - `message: string` - Текст статусу (наприклад, 'Downloading...').
  - `progress: number` - Відсоток 0-100 (якщо підтримується).
  - `total: number` - Загальна кількість кроків (для інкрементальних оновлень).
  - `current: number` - Поточний крок.
  - `indeterminate: boolean` - Крутячийся лоадер (без відсотків).
  - `start: boolean` - Початок нової сесії прогресу.
  - `stop: boolean` - Закінчення та приховування прогресу.

- **Відповідь**: Зазвичай `undefined` або стан прогресу.
- **Приклад**:
  ```javascript
  yield { type: 'progress', message: 'Starting...', start: true };
  for (let i = 0; i < 10; i++) {
    // Симуляція роботи
    await sleep(100);
    yield { type: 'progress', current: i + 1, total: 10, message: `Processing ${i + 1}/10` };
  }
  yield { type: 'progress', stop: true };
  ```
- **Примітки**: Адаптер повинен реалізовувати прогрес (наприклад, через `cli-progress` у CLI). Не блокує; продовжуйте генерувати інші інтенції.

### 3. `show` - Логування виводу

Логує повідомлення в консоль/UI. Обробляється `adapter.showIntent(intent)`.

- **Призначення**: Виведення інформаційних, налагоджувальних або попереджувальних повідомлень без запитів.
- **Обов'язкові властивості**:
  - `type: 'show'`
- **Необов'язкові властивості**:
  - `message: string` - Текст логу (перекладається через `t`).
  - `level: string` - 'info' (за замовчуванням), 'error', 'warn', 'debug', 'success'.
  - `data: any` - Додаткові дані (наприклад, об'єкт для структурованого логування).
  - `timestamp: boolean` - Включати часовий штамп (за замовчуванням: true в більшості адаптерів).

- **Відповідь**: `undefined`. Не блокує.
- **Приклад**:
  ```javascript
  yield { type: 'show', message: 'Installation started', level: 'info' };
  if (error) {
    yield { type: 'show', message: error.message, level: 'error' };
  }
  ```
- **Примітки**: Для простих випадків використовуйте `adapter.console` безпосередньо, але інтенції дозволяють адаптер-специфічне форматування (наприклад, кольори в CLI).

### 4. `render` або `renderComponent` - Рендеринг UI

Рендерить компонент або статичний контент. Обробляється `adapter.render(component, props)`.

- **Призначення**: Відображення кастомних елементів UI (наприклад, таблиці, графіки) в підтримуваних адаптерах (наприклад, web/TUI).
- **Обов'язкові властивості**:
  - `type: 'render'` або `'renderComponent'`
- **Необов'язкові властивості** (для обох):
  - `component: string|Function` - Назва компонента (рядок) або функція JSX/фабрики.
  - `props: Object` - Дані, передані компоненту (наприклад, `{ items: [...] }`).
- **Відмінності**:
  - `render`: Для простих/статичних рендерів (наприклад, markdown, HTML-рядок).
  - `renderComponent`: Для інтерактивних/повторно використовуваних компонентів (наприклад, React-подібні).

- **Відповідь**: `undefined`. Адаптер повинен підтримувати метод `render`.
- **Fallback**: Не всі адаптери мають його; ігноруйте граціозно, якщо відсутній.
- **Приклад**:
  ```javascript
  yield {
    type: 'renderComponent',
    component: 'UserTable',
    props: { users: fetchedUsers }
  };
  ```
- **Примітки**: У CLI-адаптерах це може падати на текстовий рендеринг (наприклад, через `ink` або ASCII-таблиці). Переконайтеся, що компоненти зареєстровані в адаптері.

### 5. `result` - Відображення фінального результату

Відображає вивід робочого процесу. Обробляється `adapter.resultIntent(intent)` або fallback на консоль.

- **Призначення**: Показ успіху/помилок результатів у кінці або на етапах.
- **Обов'язкові властивості**:
  - `type: 'result'`
- **Необов'язкові властивості**:
  - `data: any` - Результат (рядок, об'єкт, масив). Автоматично JSON.stringify, якщо об'єкт.
  - `success: boolean` - True для зеленого/успішного стилювання.
  - `message: string` - Підсумковий текст.
  - `format: string` - 'json', 'yaml', 'text' (за замовчуванням: авто-виявлення).

- **Відповідь**: `undefined`.
- **Fallback**: Якщо немає `resultIntent`, друкує `JSON.stringify(data, null, 2)` або `String(data)`.
- **Приклад**:
  ```javascript
  // У кінці генератора
  return { success: true, data: { files: generatedFiles } };
  // Або проміжний
  yield { type: 'result', data: 'Partial results', message: 'Check output' };
  ```
- **Примітки**: Використовуйте `return value` для фінальних результатів; це запускає неявний успіх. Для помилок кидайте або використовуйте show.

## Обробка скасувань та помилок

- **CancelError**: Імпортується з `@nan0web/ui/core`. Кидайте для скасування (наприклад, `throw new CancelError('Aborted')` у валідаторі).
- **Виявлення**: `runGenerator` перехоплює скасування та повертає `{ cancelled: true }`. У `runApp` скасування на основному рівні виходить з циклу.
- **Поширення**: У switch-case скасування встановлюють `isThrowing = true` та викликають `iter.throw(error)`.
- **Скасування користувачем**: `askIntent` повертає `{ cancelled: true }` при ESC/виході; кидає `CancelError`.
- **Кращі практики**: Завжди перевіряйте `res.cancelled` після запитів і кидайте/обробляйте відповідно.

## Структура моделі точки входу додатка

Для верхньорівневого вибору в `runApp`:

```javascript
class AppEntryModel {
  static get commands() {
    return [
      { name: 'generate', label: 'Generate Code', command: MyGeneratorCommand },
      {
        name: 'analyze',
        label: 'Analyze Project',
        run:
          async *
          function* () {
            /* inline generator */
          },
      },
    ]
  }
}
```

- Вибір генерує `{ value: { command, ... } }`.
- Підтримує функції (інстанціювати), класи (new) або прямі `{ run: generator }`.

## Поради для розробників

- **Тестування**: Мокуйте адаптер для unit-тестів. Симулюйте yields з ручним `next()`.
- **Переклад**: Використовуйте `options.t(key)` для i18n; встановіть `adapter.t = t`.
- **Продуктивність**: Генератори асинхронні; використовуйте `await` всередині для I/O.
- **Налагодження**: Генеруйте `show`-інтенції для трасувань. Перевіряйте `res.value` для результатів інтенцій.
- **Розширюваність**: Додавайте кастомні типи інтенцій у ваш адаптер; `runGenerator` передає невідомі типи як `nextVal`.
- **Обмеження**: Без паралельних yields (однопотоковий). Для складних потоків ланцюгуйте генератори.
- **Поширені помилки**:
  - Забуття `yield` перед обробкою відповідей.
  - Необробка `cancelled` у запитах.
  - Неузгоджені `field` по інтенціях.
  - Адаптери можуть відрізнятися (CLI vs. Web); тестуйте відповідно.

Для деталей адаптерів див. `InputAdapter.js`. Якщо розширюєте `runGenerator`, зберігайте switch-case інтенцій. Звертайтеся до команди OLMUI для адаптер-специфічних деталей.
