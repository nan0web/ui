# Release v1.11.0 — Architecture Consolidation (OLMUI Orchestrator & Snapshots)

Цей мінорний реліз об'єднує і стабілізує фундаментальні архітектурні рішення (інтенції, субагенти, зліпки та маршрутизацію), закладаючи надійний фундамент OLMUI v2.0.

## 🧑‍💻 User Stories (Сценарії Користувача)

### 1️⃣ Делегування Завдань ШІ-Агентам (AgentIntent)
**Як розробник (або архітектор OLMUI)**, я хочу мати можливість прозоро делегувати складні оптимізаційні задачі автономним субагентам, щоб моя доменна логіка залишалася чистою.
- З цією метою впроваджено функцію-генератор `agent(AgentContext)`. 
- Тепер будь-який доменний аудитор (наприклад, `IconsAuditor`) може зробити `yield agent(...)` і вбудований оркестратор перехопить цю інтенцію (`AgentIntent`), передасть керування `SysBuildAgent` чи `CnaiRefactorAgent`, і поверне інстанс `AgentResponse`.

### 2️⃣ Універсальний Snapshot Auditor
**Як розробник додатків**, я хочу мати єдиний надійний інструмент верифікації UI зліпків (*Snapshot Scores*).
- Замість старих розрізнених інспекторів ми перейшли на `SnapshotAuditor.js`. Тепер він автоматично оцінює зліпки на відсутність "сміття" (NaN, undefined, [object Object]), гарантуючи відповідність стандарту **Zero-Hallucination**.

### 3️⃣ Масштабований OLMUI Orchestrator (ShellModel)
**Як розробник CLI інтерфейсів**, я хочу мати можливість розширювати команди без постійного "роздування" масивів хардкоду та `switch()` конструкцій.
- Рефакторинг `ShellModel` усуває хардкод. Роутінг тепер відбувається виключно через делегування вкладеним OLMUI моделям (композиція), використовуючи багатомовні `static UI` словники.

### 4️⃣ Оптимізація Runtime та Стабільність
**Як виконавець Node.js тестів**, я хочу миттєвих тестів та максимальної оптимізації JIT-компілятора (V8).
- Усі доменні класи очищені від антипатерну *class fields outside constructor*. Усі інтенції (ask, progress, result, show, agent) експортуються як стабільні іменовані функції без попереджень компілятора TypeScript. Тестові таймаути зменшено до 3333ms (Fail-Fast підхід).
