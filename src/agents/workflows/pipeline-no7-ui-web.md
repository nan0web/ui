# 🏭 `pipeline-no7-ui-web` (Web Adapter)

> **Призначення:** Сьомий крок конвеєра. Створення Web-інтерфейсу (Lit/React компоненти).

## Інструкція для Агента

Коли Архітектор викликає цей workflow:

1. **Перевірка передумов:**
   - Доменна модель і контракти повністю готові (Кроки 2-3).
   - Інші адаптери працюють та протестовані.

2. **Створення Web Адаптера:**
   - Створити `src/ui/web/` або `src/ui/lit/` компоненти (SFC - Single File Components).
   - Component = Pure Adapter. Компоненти є "сліпими" (Dumb Components) та виключно рендерять дані з Моделі. Жодної бізнес-логіки.
   - Стилі пишуться лише через CSS variables з Design Token Manifest (`theme.js`). Ніякого хардкоду px чи кольорів.
   - Якщо використовується Lit — застосувати декоратори для прив'язки до `IntentDispatcher`.
   - Всі тексти — тільки з `.t()` або через інтерфейс моделі.

3. **Тестування:**
   - Покрити Web-адаптер Playwright snapshots (`/web-gallery`).
   - Запустити `npm run test:all`.

4. **Перевірка:**
   - Visual regression test (`olmui-scenario-test`).
