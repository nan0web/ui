description: Повний цикл оновлення UI/UX (Safe Update & Visual Verify)
---

Цей workflow обов'язковий для будь-яких змін у візуальній частині додатків (React, HTML, CSS, i18n).

### 1. Підготовка та Аналіз

1. Перевірити поточний стан UI:
   - Запустити `pnpm dev`
   - Зробити скріншот через `browser_subagent` для "ДО" (якщо задача — виправлення).
2. Перевірити структуру імпортів:
   - Використати `list_dir` для верифікації відносних шляхів (напр. `../../utils/`).
   - Не вгадувати глибину вкладеності.

### 2. Впровадження (TDD First)

1. Якщо додається логіка (jitter, фільтри, форматування):
   - Створити `src/repro.test.js` з Red-тестом.
   - Написати код.
   - Досягти Green-статусу.
2. Якщо змінюються переклади:
   - Редагувати `data/<lang>/_/t.yaml`.
   - Запустити `npm run i18n:generate`.
   - Перевірити контент `src/i18n/*.js` через `cat`.

### 3. Верифікація (DoD)

1. **Синтаксис**: Перевірити термінал `pnpm dev` на наявність "Failed to resolve import" або "Internal server error".
2. **Візуальна перевірка**:
   - Відкрити браузер через `read_browser_page` або `browser_subagent`.
   - Перевірити:
     - Чи немає raw keys (напр. `badge.onDuty`)?
     - Чи відображаються нові іконки/елементи?
     - чи працює взаємодія (кліки, ховери)?
3. **Регресія**: Перевірити Playground, якщо він використовується, щоб переконатися, що спільні компоненти не зламалися.

// turbo 4. Видалити тимчасові тести (`src/repro.test.js`) перед комітом.
