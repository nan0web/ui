# 🏭 `pipeline-no8-ui-mobile` (Mobile Adapter)

> **Призначення:** Восьмий крок конвеєра. Створення Mobile-інтерфейсу.

## Інструкція для Агента

Коли Архітектор викликає цей workflow:

1. **Перевірка передумов:**
   - Web-адаптер (Крок 7) працює та покритий візуальними тестами (`web-gallery`).

2. **Створення Mobile Адаптера:**
   - Створити `src/ui/mobile/` компоненти.
   - Застосувати PWA (Progressive Web App) конфігурації (manifest, service workers для офлайн-сценаріїв).
   - Оптимізувати Layout для Touch-Events та Adaptive / Responsive дизайну (viewport-first).
   - Використати WebView міст для нативних API, якщо це передбачено Моделлю.

3. **Тестування:**
   - Емуляція мобільних девайсів у Playwright snapshot-тестах.

4. **Перевірка:**
   - Аудит Lighthouse на PWA-сумісність.
   - Всі локальні тести зелені.

Зафіксувати версію тегом та перейти до `/pipeline-no9-qa`.
