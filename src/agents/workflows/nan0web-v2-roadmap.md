# 🚀 NaN•Web v2: Архітектурний Реліз (Zero-Hallucination Framework)

Цей документ фіксує план впровадження нових механік та архітектурних паттернів в екосистему NaN•Web, які були визначені на основі аналізу "Ради Мудреців". Головна мета впровадження — звести до нуля можливість галюцинацій штучного інтелекту при генерації коду та забезпечити структурну досконалість кожного нового додатку.

## 1. 🗂 Протокол Ініціації (The Seed Protocol)

> _Натхненно Сократом_

- **Концепція**: Заборонено генерувати абстракції або код без попереднього "Сократівського діалогу".
- **Впровадження**:
  - Агент зобов'язаний поставити користувачу 3 критичних запитання щодо місії, сутності та базових даних майбутнього проекту.
  - На основі відповідей формується файл `seed.md` (чи `abstract.yaml`), що стає "Конституцією" для поточного пакета і забороняє відхилятися від зафіксованої місії.

## 2. 🛡 Безжальні Кордони (Contract Boundaries)

> _Натхненно Макіавелі_

- **Концепція**: Жорсткі фазові переходи за `project.md`.
- **Впровадження**:
  - Неможливість переходу до створення UI-CLI чи Web-компонентів, доки не пройдено тестування моделі (`npm run verify:domain`).
  - Перевірки стають частиною скриптів екосистеми (напр. автоматичні pre-checks перед наступним кроком генерації). Агент фізично позбавляється можливості "бігти поперед батька в пекло".

## 3. 🪞 Дзеркала Даних (Pure View Models)

> _Натхненно Сковородою_

- **Концепція**: Абсолютна відмова від логіки всередині UI.
- **Впровадження**:
  - Web/Lit/React компоненти виконують виключно функцію рендеру (`render(data)`).
  - Вся валідація, трансформація і логіка утримується в `Model-as-Schema`. UI-комплекси стають просто сліпими відбивачами правди, що міститься у DB-FS.

## 4. 🎨 Design Token Manifest

> _Натхненно Леонардо Да Вінчі_

- **Концепція**: Усунення візуальних галюцинацій Агента.
- **Впровадження**:
  - Жоден Агент не має права генерувати хардкод-кольори (напр., `#ff0000`) чи розміри (`14px`) у CSS/JS.
  - Використовуватиметься виключно словник семантичних CSS-змінних (`theme.yaml` або існуючі брендові токени `Base UI`). Наказ Агенту: "Тільки семантичні змінні".

## 5. ⚡️ Диспетчер Хвиль (CLI-Bootstrap)

> _Натхненно Ніколою Теслою_

- **Концепція**: Мінімізація контекстного шуму (Context Window Focus).
- **Впровадження**:
  - Під час точкової генерації якогось рівня, Агент не повинен бачити весь монорепозиторій чи всі системні файли. Йому надається лише абстракція базового додатку, яку він розширює.

## 6. ⚔️ Шабельні Наскоки (Micro-Generations)

> _Натхненно Іваном Сірком_

- **Концепція**: Дроблення великих завдань на атомарні, швидкі удари.
- **Впровадження**:
  - Замість "напиши додаток", використовуватимуться інкрементальні кроки-workflow (напр., `npx nan0-forge`):
    - Крок 1: Генерація Моделі ➔ Збереження.
    - Крок 2: Генерація Тесту ➔ Перевірка ➔ Збереження.
  - Агент викликається лише для дрібних, конкретних задач, що мінімізує простір для фантазій.

## 7. 🏗 Blueprint Engine (Base Templates)

> _Натхненно Стівом Джобсом_

- **Концепція**: Усунення ефекту "білого аркушу".
- **Впровадження**:
  - Заведення директорії `blueprints/` (або ж використання існуючих шаблонів `.agent/templates/`).
  - При створенні нового додатку Агент копіює готовий шаблон (Blueprint) базової структури і займається лише підстановкою даних із `Model-as-Schema`.

## 8. 🛠 Контракт Зварювання (Interface Welding)

> _Натхненно Борисом Патоном_

- **Концепція**: Нерозривність коду та його верифікації.
- **Впровадження**:
  - Під час генерації будь-якого візуального компоненту (Lit, React, CLI меню), Агент **зобов'язаний** одразу, в тому ж потоці, згенерувати Snapshot-тест (чи E2E мок). Компонент без згенерованого тесту вважається невідвалідованим (браком).
