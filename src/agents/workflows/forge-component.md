# `forge-component` Workflow

> **Призначення:** Забороняє мовній моделі думати над архітектурою чи хардкодити стилі. Робить генерацію компонентів атомарною, передбачуваною та гарантує 100% покриття E2E/Snapshot тестами одразу під час написання.

## 0. Ініціація від Користувача

Користувач викликає `/forge-component [Назва]`, опціонально передаючи контекст абстрактної моделі або `theme.yaml`.

## 1. Zero-Context Focus (Диспетчер Хвиль)

- **ЗАБОРОНЕНО** читати інші компоненти чи системні файли під час генерації (щоб уникнути змішування логіки чи "фантазій").
- Якщо користувач не передав `theme.yaml` чи словник змінних локального пакета, використовуй стандартні семантичні змінні `var(--surface-color)`, `var(--text-primary)`, `var(--spacing-md)`.

## 2. Zero-Hardcode (Тупий Компонент)

- Компонент (Lit, React, CLI меню) **має тільки рендерити** дані.
- Жодних обчислень статусів, жодних тернарних операторів для перекладів!
- Отримує переклад через `db-fs` або в готовому вигляді через `props.data`.
- Усі розміри (`px`, `rem`) та кольори (`#hex`) **ЗАБОРОНЕНІ** в CSS/Inline. Використовуй тільки Token Variables (`var(--)`).

## 3. Interface Welding (Зварювання)

Створивши компонент `src/ui/[Name].js(x)`, ти **ЗОБОВ'ЯЗАНИЙ** одразу зробити наступне:

1. Зареєструвати його у `play/` пісочниці пакета (вивести його на екран).
2. Створити до нього фейкові дані в `play/data/uk/index.yaml` (якщо вони необхідні для рендеру).
3. Згенерувати `[Name].play.js` (або `snapshot.test.js` запис) для увічнення його верстки.

_Результатом виконання цього Workflow є готовий компонент, який на 100% інтегрований у внутрішню екосистему та одразу має візуальний юніт/снепшот-тест._
